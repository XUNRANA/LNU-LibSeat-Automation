### 🚀 v2.8.0 核心特性说明

本次版本迎来**终极性能重构与时序压榨**，彻底解决由于学校服务器时间刷新或 API 网络延迟导致的验证码误判与响应过慢问题，将抢座速度推向毫秒级的物理极限！

#### ⚡ 终极时序与极限提速
*   **12 秒完美宽限期**：将验证码的预加载时间大幅提前至前 `12` 秒（即 `06:29:48`），完美吸收早高峰期间第三方 API 的一切算力延迟，确保验证码坐标稳稳送达。
*   **毫秒级闪电卡点**：重构等待时序，精准设定为在 `06:29:59` 完成验证码文字点击，并在 `06:29:59.970`（提前 30ms 补偿网络延时）瞬间完成提交，彻底规避 `06:30:00` 的服务器端验证码强刷新机制！
*   **10秒防误杀机制**：将高峰期学校服务器的 `click_captcha_confirm` 响应超时忍耐度从 2 秒拉长至 12 秒，杜绝由于服务器卡顿导致正确验证码被误判报错的冤假错案。
*   **0 延迟登录**：彻底移除所有不必要的“拟人化”输入延迟，让账号密码输入到点击登录的速度从原先的 8 秒骤降至 1 秒以内。

#### 🔍 毫秒级日志与时序诊断
*   **核心日志毫秒化**：抛弃传统的秒级日志格式，无论是控制台还是 GUI 界面，所有日志精确显示到毫秒（如 `18:00:59.970`），让所有网络延迟与代码执行耗时无所遁形，便于极限性能调优。

#### 🏗️ 架构重构：单会话深度尝试策略
*   **原子化一气呵成**：重构验证码点击模块为 `fire_captcha_blitz`，将点选所有文字与提交“确定”并为一次连续的 JS 操作，将 Selenium 通信开销从数百毫秒压缩至 50 毫秒内。
*   **深度重试**：支持逐个尝试多达 10 个优先级座位，针对每个座位提供最多 10 次验证码破解机会（合计最高达 100 次高频尝试）。

#### 🔐 验证码双引擎与防检测机制
*   **商业级图鉴 API**：全新集成 [TTShiTu（图鉴）](http://api.ttshitu.com) 商业级验证码识别 API，支持 1~4 字复杂点选验证码，识别更精准。图像智能拼接上传，云端解析坐标后全自动精准点击。
*   **本地 ddddocr 兜底**：图鉴 API 出现波动时瞬间无缝降级至本地的 ddddocr 识别引擎（检测+分类双引擎），提供双重保障。识别失败自动刷新重试。
*   **高级反检测盾**：自动抹除浏览器自动化特征 (`navigator.webdriver`)、自定义 User-Agent、禁用 `AutomationControlled` 标记，完美规避系统检测。模拟真人打字速度输入，丝滑登录。

#### 🛡️ 完美防休眠与容灾机制
*   **系统级挂机守护**：完美防休眠！自动调用系统底层唤醒权限，支持长达 10 小时以上的无人看守倒计时抢座。断网、系统休眠等一切问题均不再发生。
*   **网络与维护应对**：系统维护期间自动检测，遇到网络异常自动尝试恢复连接，保障长时间挂机的稳定性。

#### 🎥 全程无死角录屏与监控
*   **内置独立录屏**：新增全新录屏模块，通过浏览器底层实时抓帧实现全程画面高清捕捉。
*   **静默模式支持**：完美支持 Headless（无头/后台静默）模式，抢座全流程将被自动录制为 `.mp4` 视频并保存在 `logs/recordings/` 目录下，方便随时复盘抢座实况。

#### 🖥️ GUI 界面与底层升级
*   **10 大优先座位扩容**：界面中的优先座位输入框从原来的 4 个直接扩展至 **10 个**（分两行直观展示），为你提供极其充足的备用选项，一网打尽所有心仪座位。
*   **核心引擎强化**：引入了 `opencv-python`、`numpy`、`mss`、`Pillow` 以及 `requests` 等强大的底层扩展依赖库，为图像高级处理、屏幕录制及云端通信提供最坚实的基础。
*   **浏览器环境持久化**：全局 `stop_event` 支持优雅停止，移除临时目录依赖，并实现浏览器配置文件持久化。

## 📖 新手小白必读！（保姆级教程）

打开软件后，你需要正确填写以下信息才能顺利抢座：

- 🎯 **目标配置**：选好你想去的校区和自习室，并填上最多十个你最心仪的座位号（按优先级排列）。
- 👤 **账号时间**：填写你的学号和密码（注意：如果没有修改过，初始密码为 `000000`）。

> ⚠️ **重中之重**：抢座的 "左区间（开始时间）" 你可以填"现在"或者整点（如：`9:00`），但抢座的 **"右区间（结束时间）" 必须填整点**，比如 `15:00`，否则学校系统是不认的！

- 📬 **成功通知**：填写你的邮箱地址，任意邮箱都行（只要邮箱存在，抢座成功就会发通知提醒你）。
- ⚙️ **执行模式**：选择"立即执行"或"定时执行"。
  - **定时执行提醒**：如果设置的预约时间大于当前时间，则认为是今天抢座；如果小于当前时间，则默认是在倒计时等待明天的抢座！
  - **卡点启动说明**：系统会在预约整点前 60 秒自动启动浏览器完成登录和预热，到预约时间前 10 秒触发验证码预分析，整点准时提交！
  - **关于静默开关**：如果你关闭了"后台静默运行"开关，系统在抢座时会光明正大地弹出一个真实的浏览器窗口，你可以亲眼目睹整个自动化抢座的详细操作流！

## 🏛️ 辽大图书馆预约规则！

> 为了避免因为一时疏忽被拉进黑名单，抢座成功后请务必遵守学校官方的签到规则：

**一、基本信息**
- 线上登录地址：`libseat.lnu.edu.cn`
- 微信入口：辽宁大学图书馆公众号 → 服务 → 座位预约
- **初始密码**：`000000`（如需修改可能通过校园网至 `opac.lnu.edu.cn` "我的图书馆" 修改）

**二、千万勿触碰的红线红区（核心规则）**
1. ✅ 预约成功后，**必须到馆刷校园卡签到**，一旦未按时签到就会被认为违约！
2. ✅ 签到时间范围：**允许提前 30 分钟签到，最晚不能超过你预约时间之后的 30 分钟**。
3. ✅ 如果遇突发情况不能去：请务必提前登录系统手动**取消预约**。
4. 🚫 **连续或 7 天内累计 3 次违约，系统将自动把你拉入黑名单，停权 7 天！！！**
5. ⏰ 系统每天放座时间：**每日 6:30** 开始放位。
6. 📊 每日最长配额：每日预约最大 3 次，每次选座时间最长跨度只能是 **6 小时**（所以你可以 9:00-15:00 和 15:00-21:00 分两次抢），每天最多只能取消 3 次。
