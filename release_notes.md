### 🚀 v2.7.0 更新内容
本次版本在 `v2.6.0` 基础上，**全面重构抢座架构**，引入图鉴 API 验证码识别、浏览器录屏，并大幅简化执行策略。

- **🏗️ 架构重构：单会话深度尝试策略**
  - 移除旧版「多轮浏览器重启」机制，改为**单浏览器会话**，不再强制重启。
  - 10 个优先级座位逐个尝试，每个座位最多 10 次验证码机会（合计最多 100 次尝试）。
  - 多账号并发时自动分配窗口布局（`slot_index` / `slot_total`），互不干扰。

- **⏰ 三阶段精确卡点**
  - `prep_at`（提前 60s）：启动浏览器、登录、进入目标自习室。
  - `pre_fire_at`（提前 10s）：锁定座位，触发验证码弹窗并预分析。
  - `fire_at`（准时）：点击验证码「确定」并提交预约。

- **🔐 图鉴 API 验证码识别**
  - 新增 `core/captcha_api.py`，集成 [TTShiTu（图鉴）](http://api.ttshitu.com) 付费点选验证码识别。
  - 将提示文字图与点击大图竖向拼接后上传 API，解析返回坐标并自动点击。
  - 图鉴 API 优先使用，失败时自动降级到本地 ddddocr 引擎。

- **🎥 浏览器全程录屏**
  - 新增 `core/screen_recorder.py`，通过 `driver.get_screenshot_as_png()` 实现浏览器内录屏。
  - 支持 headless 模式，录制 MP4 保存到 `logs/recordings/`。

- **🖥️ GUI 优先座位扩展**
  - 优先座位输入框从 4 个增加到 **10 个**，分两行显示。

- **📦 新增依赖**
  - `opencv-python`、`numpy`、`mss`、`Pillow`、`requests`。

### 🚀 v2.6.0 更新内容
本次版本在 `v2.5.0` 基础上，重点升级了**定时抢座时序**和**浏览器重启容灾机制**，同时继续保持配置文件脱敏分发。

- **⏰ 定时模式时序升级**
  - 定时模式改为：**提前 90 秒启动浏览器、登录并预进入目标自习室**。
  - 到 `fire_at` 时刻才真正开始：**点座 → 点击「立即预约」→ 识别点选验证码 → 按顺序点字 → 点击确定**。
  - 严格模式默认 `6:30` 开抢，自定义模式支持任意指定时刻（例如 `13:00`、`11:25`）。

- **🔁 浏览器硬重启策略**
  - **立即模式**与**定时模式**统一采用：**每轮 5 分钟，最多 6 轮**。
  - 一旦单轮达到截止时间，**无论当前卡在登录、选座、验证码还是确认步骤，都会强制关闭浏览器并从头启动下一轮**。
  - 定时严格模式默认整体抢到 `7:00`；自定义定时模式默认整体抢到 `fire_at` 后 30 分钟。

- **🔐 点选验证码增强**
  - 支持 **1~4 字**目标串。
  - 强制按目标串顺序点击对应文字坐标。
  - 识别失败自动刷新并重试（最多 5 次）。

- **🔒 配置文件安全分发**
  - 发布包中的 `config.py` 保持为**干净模板**，不会夹带本机账号、密码、邮箱等个人信息。

## 📖 新手小白必读！（保姆级教程）

打开软件后，你需要正确填写以下信息才能顺利抢座：

- 🎯 **目标配置**：选好你想去的校区和自习室，并填上最多十个你最心仪的座位号（按优先级排列）。
- 👤 **账号时间**：填写你的学号和密码（注意：如果没有修改过，初始密码为 `000000`）。

> ⚠️ **重中之重**：抢座的 "左区间（开始时间）" 你可以填"现在"或者整点（如：`9:00`），但抢座的 **"右区间（结束时间）" 必须填整点**，比如 `15:00`，否则学校系统是不认的！

- 📬 **成功通知**：填写你的邮箱地址，任意邮箱都行（只要邮箱存在，抢座成功就会发通知提醒你）。
- ⚙️ **执行模式**：选择"立即执行"或"定时执行"。
  - **定时执行提醒**：如果设置的预约时间大于当前时间，则认为是今天抢座；如果小于当前时间，则默认是在倒计时等待明天的抢座！
  - **卡点启动说明**：系统会在预约整点前 60 秒自动启动浏览器完成登录和预热，到预约时间前 10 秒触发验证码预分析，整点准时提交！
  - **关于静默开关**：如果你关闭了"后台静默运行"开关，系统在抢座时会光明正大地弹出一个真实的浏览器窗口，你可以亲眼目睹整个自动化抢座的详细操作流！
