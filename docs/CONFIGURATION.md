## 一、账号设置 (USERS)

```python
USERS = {
    "你的学号": {
        "password": "你的密码",
        "time": {"start": "9:00", "end": "15:00"}
    },
    "第二个学号": {
        "password": "密码",
        "time": {"start": "15:00", "end": "21:00"}
    }
}
```

- **并发机制**：系统根据字典 keys 自动孵化相应数量线程。
- **分时段占全天**：两个号抢同一座位、不同时段，实现全天覆盖。

## 二、目标场馆与座位 (TARGET & PREFER)

```python
TARGET_CAMPUS = "崇山校区图书馆"
TARGET_ROOM = "三楼智慧研修空间"
PREFER_SEATS = ["185", "186", "187", "188", "189", "190", "191", "192", "193", "194"]
```

- **校区**：`"崇山校区图书馆"` 或 `"蒲河校区图书馆"`。
- **自习室**：必须与网页端显示完全一致。程序内置双校区全部 20 间自习室座位索引。
- **PREFER_SEATS**：留空 `[]` 表示不指定首选，直接随机扫描全自习室。填了则按顺序优先尝试 → 失败后自动兜底扫描剩余座位。

### 可用的自习室列表

**崇山校区（8间）**：二楼书库北、二楼书库南、二楼背诵长廊、三楼智慧研修空间、三楼理科书库、四楼北自习室、四楼南自习室、四楼自习室406

**蒲河校区（12间）**：三楼走廊、4楼阅览室、四楼走廊、5楼阅览室、五楼走廊、6楼阅览室、六楼走廊、704、706、707、708、七楼走廊

## 三、休眠控制与定时

```python
WAIT_FOR_0630 = True
```

- **`True`**：定时模式。程序提前 30s 启动浏览器 → 提前 6s 锁定座位 → 整点准时提交。
- **`False`**：立即模式。直接开抢，锁住就提交。

定时目标时刻可在 GUI 中配置（严格模式默认 6:30，或自定义时刻）。

## 四、图鉴 API 设置

```python
FORCE_API_ALWAYS = False
```

- **`True`**：全天使用图鉴 (TTShiTu) 商业 API 识别验证码，每个座位最多 **5 次**重试。
- **`False`**：仅每日 **6:30-6:35** 高峰期使用 API，其余时间使用本地 ddddocr（每座位最多 **10 次**重试）。

> 图鉴 API 为付费服务，凭据内嵌在程序中。可在 GUI 中一键切换。

## 五、邮件通知

```python
RECEIVER_EMAIL = "你的邮箱@qq.com"
SMTP_USER = ""
SMTP_PASS = ""
```

- **`RECEIVER_EMAIL`**：接收通知的邮箱。**填这一项即可收到邮件！**
- **`SMTP_USER` / `SMTP_PASS`**：留空使用内置发件邮箱；填入则使用自定义邮箱。

## 六、浏览器等进阶设定

```python
BROWSER = "edge"
DRIVER_PATH = ""
WEBDRIVER_CACHE = ""
LOG_LEVEL = "INFO"
LOG_DIR = "logs"
```

- **`BROWSER`**：`"edge"` 或 `"chrome"`。
- **`DRIVER_PATH`**：手动指定 WebDriver 路径（自动下载失败时使用）。
- **`LOG_LEVEL`**：`"DEBUG"` / `"INFO"` / `"WARNING"` / `"ERROR"`。
- **`LOG_DIR`**：日志输出目录。
  - `logs/lnu_seat.log` — 全量日志
  - `logs/lnu_seat_{账号}.log` — 分账号日志
  - `logs/sessions/<时间戳>_<账号>/` — 每次抢座的专属文件夹
